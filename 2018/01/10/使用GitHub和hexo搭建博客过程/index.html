使用GitHub和hexo搭建博客过程

发表于 2018-01-10 | 分类于 教程 | 阅读次数:

其实网上已经有好多这样的教程了，然而我还是要写( • ̀ω•́ )✧
因为搭建这博客折腾了我两天啊(╯‵□′)╯︵┻━┻。
希望能给以后的同学多一篇参考资料吧(￣▽￣)／

本教程以windows平台的电脑为主，macOS平台也可以参考一下，因为过程都是一样的

环境搭建

安装Node.js：http://nodejs.cn/
安装Git： https://git-scm.com/downloads
安装这两个软件都很简单，下载安装就好，实在不会就自己搜索一下吧

安装完Git配置一下环境变量：https://jingyan.baidu.com/article/fec4bce271601ff2618d8be3.html
【注意】：添加Path时，如果里面已经有东西了，要先添加英文状态的分号;再添加自己的

搭建本地Hexo项目

初始化项目

随便新建个文件夹，例如我在g盘建了个testBlog文件夹，在文件夹里按着shirt键同时按鼠标右键，选择“在此处打开命令窗口”

  • 输入命令，按回车键执行，等待下载完…

    npm install -g hexo-cli


如果出现红色ERR错误就执行命令 npm uninstall -g hexo-cli 卸载一下重新执行上面的命令即可。

  • 完成后继执行命令（每一行按一次回车键，每行执行完再下一行，不急）

    hexo i
    hexo g
    hexo s

执行完最后一行命令会是这样的↓

打开浏览器，访问地址：

localhost:4000

到这里说明你成功啦 ヽ(ﾟ∀ﾟ)ﾒ(ﾟ∀ﾟ)ﾉ

使用NexT主题

话说隔壁的博客好好看啊简洁大方，我们也就可以有啊
我们选择NexT主题为例子哈
稳定版都在这里：https://github.com/iissnan/hexo-theme-next/releases
或者用当前(写博客时)最新的稳定版：

https://codeload.github.com/iissnan/hexo-theme-next/zip/v5.1.4

解压所下载的压缩包，得到名为 hexo-theme-next-5.1.4 的文件夹，把名称更改为 next 放在 themes 目录下

打开 站点配置文件， 找到 theme 字段，并将其值更改为 next （原来为 landscape，即我们上面打开localhost:4000看到的默认主题 ）
【 站点配置文件 就是项目根目录下的_config.yml文件】
【所有配置文件设置冒号后面都有空格，注意注意】(oﾟ▽ﾟ)o

执行命令(一如既往地一行行执行，如无特殊说明，下面所有命令行都是)
PS：任何时候动了_config.yml文件 都要先执行 hexo clean 再执行 hexo g之类的命令

hexo clean
hexo g
hexo s

如果你的命令行窗口是这样的，只要同时按 Ctrl键和C键 ，然后输入字母 y ，即可输入命令

执行完上面3行命令得到的也是上面↑ 这个界面，因为做的是同样的事情嘛
一样的步骤，打开浏览器，访问地址：

localhost:4000

主题换成功了哈

也可以选择自己喜欢的主题哈，打开主题目录下的_config.yml文件设置即可，当前我们用的是Muse，改了记得执行那3行命令哦(づ｡◕ᴗᴗ◕｡)づ

当然你也可以自己改源码，甚至自己写主题哈︿(￣︶￣)︿

NexT主题详细设置参考：
http://wangwlj.com/2017/09/09/blog-opti/

创建GitHub仓库

博客已经完成一半了，再坚持一下(๑•̀ㅂ•́)و✧
去GitHub注册个账户：https://github.com/
英文不好的同学慢慢来，注册很简单的就不说了。

注册完登录，点击按钮创建新仓库

注意要设置的地方↓ 设置完成后点击绿色按钮创建就好。

创建完毕，点击setting

页面往下滚，找到 GitHub Pages 选择 master 点就save

点击save后页面可能会滚到顶，没事，滚回去找回 GitHub Pages ，点击 choose a theme 随便选个主题，不用挑反正用不上，等下要覆盖掉的(･ω･)ﾉ

就选第一个吧，绿色按钮确定选择

跳转页面就往下滚，找到绿色按钮 commit changes，点击就好

点击GitHub Pages的那个地址(上面的图片有，就在GitHub Pages下面浅蓝色区域那个地址)
例如我的是：http://luogjy.xyz/Luogjy.test.github.io/ （我的有xyz这么奇怪是因为我之前绑定了域名，这个不影响，用自己的就行）
看到刚才选择的主题就成功了

本地Hexo项目与GitHub仓库关联

快了快了，再坚持一下下(๑•̀ㅂ•́)و✧
回去刚才的仓库，复制一下仓库地址

设置 站点配置文件 _config.yml ，仓库地址就粘贴在 repository那里

执行命令，第一行命令是下载hexo-deployer-git，第二行命令用来上传代码到刚才建的仓库

npm install hexo-deployer-git –save
hexo d

哈哈出错了，因为git还不知道你要提交到那里，设置一下就好了

执行两个命令如图

hexo d

成功了是这样的

在打开GitHub Pages的那个地址看看是不是成了
(上面的图片有，就在GitHub Pages下面浅蓝色区域那个地址)
例如我的是 http://luogjy.xyz/Luogjy.test.github.io/

这样就完成了本地Hexo项目与GitHub仓库关联啦

怎么发布博客

首先执行命令安装个插件哈，这个插件是用来发布博客时带上图片的
执行命令

npm install hexo-asset-image –save

在站点配置文件 _config.yml 找到 post_asset_folder 选项，设置为true

执行命令

hexo new “我的第二篇博客”

名字随便起哈，为什么我叫它第二篇？因为默认已经有一篇在上面啦，你也可以删掉哈

这样就成功啦哈哈

生产的博客文件在_posts文件夹下，是.md文件哈，就是用Markdown语法写的，很简单的不用怕，网上资料一大堆哈
另外有一个同名文件夹，博文的图片放在这个文件夹里就好，需要的时候使用Markdown语法引用。

然后你就可以在里面愉快地写啦，一篇博客一个.md文件哦

打开地址看看，如无意外就成功了哈哈

一些错误处理

  • 博客搭建完第n天以后发布博客，执行命令 hexo d 后出现以下错误

    解决方案：
    执行命令
    git config –global http.sslVerify false
    然后重新执行hexo d 即可

  • 已经成功执行过npm install hexo-asset-image –save命令，而且正确地引入了图片以后，本地通过http://localhost:4000/ 预览时看不到引入的图片

    解决方案：
    发布上去GitHub上再看看，一般已经成功看到图片了。
